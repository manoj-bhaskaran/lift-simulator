# Conventional Commits Quick Reference

## Format

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

## Types & Examples

### feat (new feature, bumps MINOR version)
```bash
git commit -m "feat(logging): add comprehensive logging to GlobalExceptionHandler"
git commit -m "feat(api): add endpoint for retrieving system metrics"
```

### fix (bug fix, bumps PATCH version)
```bash
git commit -m "fix(validation): correct email validation regex"
git commit -m "fix(api): handle null pointer in configuration service"
```

### chore (maintenance, NO version bump)
```bash
git commit -m "chore(deps): update Spring Boot to 3.2.2"
git commit -m "chore(build): configure Maven resource filtering"
```

### docs (documentation, NO version bump)
```bash
git commit -m "docs(readme): update installation instructions"
git commit -m "docs(api): add OpenAPI documentation"
```

### refactor (code restructuring, NO version bump)
```bash
git commit -m "refactor(service): extract validation logic to separate class"
```

### perf (performance improvements, NO version bump)
```bash
git commit -m "perf(database): add index on lift_system.system_key column"
```

### test (tests, NO version bump, hidden from CHANGELOG)
```bash
git commit -m "test(controller): add integration tests for exception handlers"
```

### BREAKING CHANGE (breaking change, bumps MAJOR version)
```bash
git commit -m "feat(api): redesign configuration API

BREAKING CHANGE: Configuration endpoints moved from /api/config to /api/v2/config.
Clients must update their API calls to use the new endpoints."
```

## Multi-line Commit with Details

```bash
git commit -m "feat(logging): add comprehensive logging to GlobalExceptionHandler

Add SLF4J-based logging for all exception handlers with appropriate log levels:
- ERROR: Generic exception handler with full stack traces
- WARN: Malformed JSON requests
- INFO: Business exceptions (404, 400, 409)
- DEBUG: Validation errors

Benefits:
- Enables debugging of production issues
- Provides audit trail of exceptions
- Improves system observability

Closes #123"
```

## Scopes (Examples)

- `logging` - Logging infrastructure
- `api` - REST API endpoints
- `validation` - Validation logic
- `database` - Database/persistence layer
- `frontend` - Frontend/UI changes
- `deps` - Dependencies
- `build` - Build configuration
- `docs` - Documentation
- `test` - Tests
- `config` - Configuration

## Quick Workflow

1. **Make changes**
2. **Stage files**: `git add .`
3. **Commit with conventional format**: `git commit -m "feat(scope): description"`
4. **Run release**: `npm run release`
5. **Enhance CHANGELOG if needed**
6. **Push**: `git push --follow-tags`
